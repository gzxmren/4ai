# 给中国历史复习 Agent 的沟通文档

> 本文档源于 F2 地理期末考试复习的实际经验。地理科复习材料覆盖率 95%，但考试仍暴露了深度、广度、难度三个层面的显著偏差。以下四项核心经验，可直接迁移到中国历史科的复习准备中。

---

## 核心经验一：深度问题 — 知识 ≠ 应用

### 现象

复习笔记把概念讲清楚了，但考试考的是"用概念分析数据"。

| 复习笔记覆盖的内容 | 考试实际要求 |
|------------------|------------|
| "Intensive farming = high inputs per unit area" | 给表格（834m² 面积 / 15 个工人），要求**计算** labour per area，再判断 |
| "LDC 比 MDC 更脆弱" | 给 Table 1b 的 5 行数据，要求**选** 2 个指标并引用具体数字论证 |
| "East China = plains → arable farming" | 反过来：看到 farm size small + major output rice，推断是华南地区 |
| "Asthenosphere 是软流圈，可流动" | 填空题直接填词（OK 够用） |

### 关键发现

同是概念，**考 recall 的题笔记够用**（如填空、选择），但**考应用/分析的题笔记深度不够**。

### 给历史科的转移

不能只让学生"知道"鸦片战争发生在 1840 年、有三个条约，而要训练：

| 考试题型 | 笔记需要达到的深度 |
|---------|----------------|
| 列举/列出（recall） | 列表即可 ✅ |
| 描述事件经过（describe） | 时间 + 地点 + 人物 + 过程 ✅ |
| **解释原因/分析影响（explain/analyze）** | 需要**因果链**、**多因素分析框架** ⚠️ |
| **比较异同（compare）** | 需要**对比表格**（维度对齐）⚠️ |
| **评价人物/事件（evaluate）** | 需要**正反两面论据** ⚠️ |
| **史料分析（source analysis）** | 需要**解读方法和范例** ❌ 最常缺失 |

> **检查方法**：拿起试卷，标出每道题的认知层次（recall / describe / explain / compare / evaluate），然后对照复习笔记看深度是否匹配。不匹配的地方就是缺口。

## 核心经验二：广度问题 — 容易漏掉"过程性技能"

### 现象

复习笔记覆盖了所有概念性知识（95% coverage），唯独漏了一类——**不是"知识点"，而是"怎么用知识点"的技能**。

地理科的具体案例：**方位角计算（bearing）**。
- 复习笔记完整覆盖了等高线、剖面图、V.E.、坡度计算
- 但 **whole circle bearing 和 reduced bearing** 彻底没讲
- 原因：它属于"地图实操技能"，在按单元整理笔记时容易被忽略

### 对历史科的警示

历史科中也有类似的"过程性技能"，按重要程度排列：

| 优先级 | 过程性技能 | 检查方法 |
|-------|-----------|---------|
| 🔴 高 | **史料分析**：判断立场、可信度、一手/二手 | 试卷中是否有原文 + 分析题 |
| 🔴 高 | **时间线排列**：事件先后顺序 | 试卷中是否有排序题 |
| 🟡 中 | **因果链推导**：从事件 A 到结果 B 的逻辑链路 | 试卷中是否有"解释为什么" |
| 🟡 中 | **对比框架**：两个事件/朝代按多维度对齐 | 试卷中是否有"比较"题 |
| 🟢 低 | **地图识读**：疆域变化、路线图 | 试卷中是否有地图类资料题 |

> **检查方法**：给复习笔记做一个分类统计——有多少是"事实陈述"（XX 年发生了 XX），有多少是"方法框架"（如何分析史料、如何比较两个朝代）。如果后者占比过低，就是广度缺口。

## 核心经验三：难度问题 — 模拟题必须匹配真题结构

### 现象

地理复习的模拟题（day3_comprehensive.html）与实际考试严重脱节：

| 对比项 | 模拟题 | 实际考试 |
|-------|-------|---------|
| 总分 | 20 分 | 100 分 |
| 题型种类 | MCQ + 2 道结构题 | 5 种题型 |
| 认知层次 | 纯概念回忆 | 含资料分析、计算、推理 |
| 需引用数据 | 无 | 每道资料题都要求 "justify with evidence" |
| 实操部分 | 无 | 画剖面图、计算 V.E./坡度/方位角 |

### 后果

学生在模拟题上能拿高分，但真实考试完全不同的题型和难度，直接导致准备不足。

### 给历史科的转移

拿到试卷后，先建一个 **"题型-分值-认知层次"矩阵**：

```
题型             分值   认知层次        模拟题中是否有？
─────────        ──    ──────────      ─────────────
MCQ              10    recall          ✅
Fill in blanks   10    recall          ✅
Matching         6     recall          ❌ 加一个
True/False       9     recall          ❌ 加一个
Data-based Q1    19    analyze+apply   ❌ 需要设计（给史料/图表）
Data-based Q2    17    understand+explain ❌ 需要设计
Data-based Q3    9     analyze+compare ❌ 需要设计
Data-based Q4    20    apply+calculate ❌ 需要设计
```

原则：**考试有什么题型，模拟题就必须有什么题型**。分值比例也尽量接近。

## 核心经验四：流程问题 — 先读试卷，再做计划

### 原本的做法

```
整理复习笔记（按单元）→ 制定三天计划 → 最后出模拟题
                                                       ↑ 缺口发现太晚
```

### 应该的做法

```
先花 15 分钟读试卷照片
    ↓
了解：考什么题型？考多深？有哪些技能要求？
    ↓
然后制定复习计划（针对性地分配时间）
    ↓
边整理笔记边对照试卷需求调整深度
    ↓
模拟题与真题同构
```

### 具体操作建议

| 时间点 | 做什么 |
|-------|-------|
| 拿到试卷后第一个 15 分钟 | 通读所有题目，记录题型、分值、认知层次 |
| 制定复习计划时 | 给"过程性技能"（史料分析、对比框架）预留 >30% 的时间 |
| 每整理完一个单元笔记 | 回头对一下试卷中相关的题：我笔记的深度够不够回答这道题？ |
| 出模拟题时 | 用"题型-分值-认知层次矩阵"逐一核验 |

## 附：工作流速查模板

### 拿到试卷后的执行清单

```
[ ] 并行读取所有试卷照片（一次性 read_file）
[ ] 记录试卷结构：题型 / 分值 / 范围
[ ] 读取所有现有复习材料
[ ] 制作"三栏映射表"：
    ┌────────────┬────────────┬──────────┐
    │ 考题内容   │ 对应知识点  │ 覆盖状态  │
    ├────────────┼────────────┼──────────┤
    │ Q1 ...     │ ...        │ ✅/⚠️/❌ │
    └────────────┴────────────┴──────────┘
[ ] 三维评估：
    [ ] 内容覆盖面（有没有讲？）
    [ ] 知识点深度（够不够深？）
    [ ] 难度匹配（模拟题对不齐？）
[ ] 输出交互式 HTML（题目 + 可展开答案 + 分析表）
[ ] 一次性 git add 所有变更文件 → commit → push
```

### 交互式 HTML 模板结构

```
标题 + 考试信息
├── Section A: MCQ（每题 = 卡片 + 展开答案）
├── Section B: 配对
├── Section C: 填空
├── Section D: 判断
├── Section E: 资料题（按小问展开）
├── 总分析表（knowledge mapping）
├── 缺口汇总（橙色卡片）
└── 优势汇总（绿色卡片）
```

答案卡片设计要点：
- 默认折叠，点击展开（▶/▼）
- 对依赖图片的题给出**解题方法**，不猜答案
- 论述题提供**评分要点**（"提及 3 个影响各得 1 分"）
- 对比题用**表格呈现**

---

*本文档基于 2025-2026 F.2 Geography Final Exam 复习准备的实战经验整理。核心洞察来自：复习覆盖率 95% ≠ 考试准备充分。真正的差距在三个维度：深度（知识 vs 应用）、广度（概念 vs 技能）、难度（模拟 vs 真题）。*

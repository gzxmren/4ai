從地理到歷史：答題方法框架的遷移經驗總結

本文檔記錄了中國歷史科複習資料的「深度×廣度×難度」三維改造過程。 供西方歷史科和科學科的 agent 參考借鑑。

改造靈感來源：地理 agent 的實戰復盤（from_geo/agent_handoff_history_methodology.md） 改造成果：本目錄下的 4 個 技能框架_*.md + 認知深度索引圖.md + 模擬試卷.html（深度版）

核心理念：問題不是「缺知識」，而是「缺方法」

無論哪個科目，學生最常見的問題都不是「不知道事實」，而是不會用事實來回答更高層次的問題。

層次 學生能做的事 常見問題
recall 背出定義、日期、人名 ✅ 大部分學生能做到
describe 描述事件經過 ✅ 大部分能做到
explain 解釋原因 ⚠️ 需要引導
analyze 分析影響、趨勢 ⚠️ 需要框架
compare 比較異同 ❌ 需要方法
evaluate 評價人物/事件/政策 ❌ 最薄弱

核心洞察：翻閱任何一份複習筆記，內容幾乎都是 recall 層次。而考試中能拉開分數的題目，都在 explain→evaluate 層次。

第一步：做三維診斷（1-2 小時）

1.1 深度診斷（Depth）

拿起現有的複習資料，逐主題統計認知層次覆蓋率：

主題：XXX
  recall    ████████████ 80%  ← 筆記大部分在這
  describe  ████         20%
  explain   ██           10%  ← 考試會考的缺口
  analyze   █             5%
  compare   █             5%
  evaluate  █             5%

判斷標準： - 如果 explain+analyze+compare+evaluate 合計 < 30% → 深度嚴重不足 - 如果 evaluate = 0% → 開放題型完全缺乏準備

1.2 廣度診斷（Breadth）

開一個表格，列出該科目特有的「過程性技能」：

科目 過程性技能（可能被遺漏）
中國歷史 （已處理） 史料分析 · 因果鏈 · 對比框架 · 評價方法
西方歷史 史料分析 · 時間線 · 因果鏈 · 比較文明 · 評價歷史人物
科學 實驗設計 · 數據分析 · 圖表解讀 · 結論推導 · 誤差分析

判斷標準： - 如果某項技能在複習筆記中「完全找不到對應章節」→ 廣度缺口 - 檢查方法：搜索筆記中的關鍵詞（如「分析」「比較」「評價」「證明」）

1.3 難度診斷（Difficulty）

將模擬題和（如果有）真實試卷做對比：

維度 模擬題 真實試卷（如果有）
題型種類 幾種？ 幾種？
分值分佈 各題佔分 各題佔分
認知層次 最高到哪一層？ 最高到哪一層？
需要引用證據 是否要求？ 是否要求？

判斷標準： - 模擬題的題型 < 真實試卷的題型 → 難度不匹配 - 模擬題的最高認知層次 < 真實試卷的最高層次 → 難度不足

第二步：針對性地補充框架（2-4 小時）

診斷完之後，不要直接修補原有筆記——而是獨立創建「答題方法框架」文件。

2.1 框架文件的標準結構

每個框架文件應包含 4 個部分：

1. 方法說明（Step-by-step）
   └── 用幾步能完成這類題目？

2. 固定模板（Scaffold）
   └── 填空式的答題結構，學生可以直接套用

3. 該科目的具體案例
   └── 用真實的教材內容做範例

4. 實戰練習（附答案）
   └── 學生可以做來自測

2.2 中國歷史科的四個框架（可作為範本）

框架 核心方法 對應題型 對其他科目的可遷移性
史料分析 S-T-A-R 四問法 資料題 西方歷史可直接複用，科學科可改為「數據/實驗結果分析」
因果鏈 三步法 + 分類 解釋題 西方歷史可直接複用，科學科可改為「現象的原因和影響」
對比分析 三步法 + 模板 比較題 所有科目通用
評價 P-E-B-C 四步法 開放題 所有科目通用

2.3 關鍵：不要猜答案，給方法

每個框架文件都應該教學生「如何思考」，而不是「這道題的答案是甚麼」。

❌ 不好：  「鴉片戰爭的原因是英國想打開中國市場。」
✅ 更好：  「分析原因時，先區分遠因（長期結構性問題）和近因（直接觸發事件）。」

第三步：改造模擬題（1-2 小時）

3.1 題型-分值-認知層次矩陣

改造前後做一個對比表。以中國歷史為例：

改造前：
  填空題 20分 (recall)    時序題 10分 (recall)    配對題 10分 (recall)
  選擇題 20分 (recall)    資料題 40分 (analyze)    ← 資料題深度不足

改造後：
  填空題 20分 (recall)    時序題 10分 (understand)
  配對題 10分 (recall)    選擇題 20分 (recall/understand)
  史料分析題 40分 (analyze/compare/evaluate)
                        ↑ 加入了 compare 和 evaluate

3.2 資料題的關鍵升級

這是分值最高的部分，最容易拉開差距：

舊版 新版
提供改寫的總結性文字 提供有出處的真實史料（作者、年份、背景）
只問「指出/列出」 問「指出並引用證據」
只有 recall 層次 包含 analyze → compare → evaluate
答案簡單 答案附評分標準 + 框架引用提示

3.3 對科學科的特別建議

科學科的資料題可以類比為：

歷史資料題 科學科對應
一手史料（原始文獻） 原始實驗數據 / 觀測記錄
二手史料（學者分析） 教科書結論 / 科學家論文
S-T-A-R 分析 實驗設計評估（變量/對照/誤差/結論）
資料引用論證 用數據證明結論
評價史料可信度 評估實驗的可重複性和侷限性

第四步：製作認知深度索引圖（30分鐘）

這是最後一步，作用是讓學生知道自己現在在哪裡、要去哪裡。

4.1 格式

按主題列出每個認知層次的覆蓋情況：

主題：XXX

| 層次 | 你應該能…… | 覆蓋 | 參考 |
|------|-----------|------|------|
| recall | 說出…… | ✅ | 筆記第X頁 |
| understand | 描述…… | ✅ | |
| explain | 解釋為甚麼…… | ⚠️ | 因果鏈框架 |
| analyze | 分析影響…… | ❌ | 待補充 |
| compare | 比較…… | ⚠️ | 對比框架 |
| evaluate | 評價…… | ❌ | 評價框架 |

4.2 給出時間分配建議

recall（知道）   20% 時間 ← 學生通常花 80% 時間在這裏
understand（理解）15%
explain（解釋）   20%
analyze（分析）   20%
compare（比較）   10%
evaluate（評價）  15%

建議的閱讀和執行順序

第一步（30分鐘）：讀懂全局

1. from_geo/agent_handoff_history_methodology.md
   └── 地理科的實戰復盤，理解「深度×廣度×難度」的由來
2. agent_handoff_deliverable.md
   └── 本文，完整的方法論和執行清單

第二步（1-2小時）：做自己的三維診斷

3. 讀取你的科目的複習筆記和試題
4. 按本文第一步的方法，統計認知層次覆蓋率
5. 列出你的科目特有的過程性技能
6. 記錄缺口

第三步（2-4小時）：創建或改造框架

7. 按需複用/改編上表中的框架文件
8. 每個框架按「方法說明→模板→案例→練習」四段結構

第四步（1-2小時）：改造模擬題

9. 建立題型-分值-認知層次矩陣
10. 改造資料題，加入真實資料 + 要求引用證據 + 增加 compare/evaluate

第五步（30分鐘）：收尾

11. 製作認知深度索引圖
12. 一次性 commit 所有修改

最終提醒

  1. 不要改原有的複習筆記——學生已經習慣了，改了會造成混亂。新增框架文件和索引圖即可。
  2. 框架文件之間不要重複——每個框架只解決一類問題。
  3. 模擬題改造要保留原有的 recall 部分——recall 題是基礎，不能丟掉，只是在上層增加 analyze/compare/evaluate。
  4. 方法優先於答案——給學生「釣竿」，而不是「魚」。
